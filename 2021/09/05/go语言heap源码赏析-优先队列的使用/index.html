前置知识

堆

概念

如果有一个关键码的集合K = {k0，k1， k2，…，kn-1}，把它的所有元素按完全二叉树的顺序存储方式存储在一个一维数组中，并满足：Ki <= K2i+1 且 Ki<=K2i+2 ，则称为小堆(或大堆)。

将根节点最大的堆叫做最大堆或大根堆，根节点最小的堆叫做最小堆或小根堆。

性质

  1. 堆中某个节点的值总是不大于或不小于其父节点的值;
  2. 堆总是一棵完全二叉树。

一个很重要的点：

在二叉树(堆是特殊的二叉树)中，如果父节点的下标为k，那么如果其子节点存在，则子节点的下标为2k + 1，2k + 2.

Init()

我们从heap.Init()看起。

非常简单，里面只调用了一个down()方法。至于i为什么从n/ 2 - 1到0，我们看完down函数再来解释。

在这个函数中，传进来的参数有三个，分别是接口Interface，父节点下标，Interface长度。

通过这个函数，可以实现将传进来的父节点向下寻找到其该处于的位置。

简而言之，就是让父节点、左右节点三个节点中值最大的处于父节点的位置，在不断的交换过程中，找到一个位置，使得我们传进来的父节点处于一个合适的位置(大于其两个子节点)。

Push()

再来看看heap.Push()函数。

非常简单，调用h自己实现的Push，然后对push后的Interface进行整理。

与刚才的down相反的是，down是把元素从上往下沉淀，而up是把元素从下往上浮起。

为什么这里要使用down呢？因为这里实际上是对Inerface append了一个元素，而这个元素处于Interface的末尾，也只有这个元素是无序的，因此摆正它的位置即可。

Pop()

首先将头部元素和末尾元素进行交换，此时的头部元素是无序的，因此直接down(h, 0, n)，让此元素找到合适的位置即可，最后调用h的pop，将末尾元素弹出。

这里会想到一个问题，交换后的头部元素在找其位置时，按理说应该又会找回到了末尾元素，那么头部元素又会被交换至末尾元素了，那就达不到弹出原头部元素的目的了。

Remove()

注意，这里的参数i不是元素i，而是下标i，remove掉的是i下标处的元素。

与pop()思路大致一致，先将该下标的元素与末尾元素进行交换，然后摆正交换后的该元素的位置。

注意的点是，在push或pop中，都是从末尾开始向上找位置，而在remove中是从中间位置开始找，因此向下找一次，如果元素没有向下移动，则向上再寻找，down中最后一句return就是为此做准备的。

Fix()

Fix与remove思路一致，因为无法保证i的位置，因此先向下寻找，再向上寻找。
